# ch03：工具系统与最小 Agent Loop Spec

## 1. 背景

YuanCode 当前已经能够在终端中与 Anthropic、OpenAI 兼容后端进行流式多轮对话，但模型只能生成文本，无法读取项目、修改文件或执行命令。仅增加一组本地工具还不够：Provider 必须能发送工具定义并解析工具调用，会话必须保存工具调用与执行结果，主流程还必须在一次用户请求中持续完成“模型决策—工具执行—结果回传—模型继续回答”。

本阶段交付工具统一契约、工具注册与发现、六个核心工具、终端询问工具，以及可运行的最小 Agent Loop，使 YuanCode 从纯聊天助手升级为具备基础代码库操作能力的 Coding Agent。

## 2. 目标用户

- 在本地终端中使用 YuanCode 阅读、搜索和修改代码的开发者。
- 希望让模型执行项目命令，并在同一轮对话中基于结果继续推理的开发者。
- 后续需要接入更多本地工具、MCP 工具或子 Agent 工具的 YuanCode 维护者。

## 3. 能力清单

- 用统一工具契约描述工具名称、用途、输入结构、执行类别和执行结果。
- 用注册表集中完成工具注册、查找、稳定排序和 Provider schema 转换。
- 支持延迟工具发现，避免所有可选工具长期占用模型上下文。
- 允许模型按区间读取文本文件，并获得带行号的内容。
- 允许模型写入文本文件，并在需要时创建父目录。
- 允许模型通过唯一文本匹配精确编辑已有文件。
- 允许模型通过 Bash 执行命令，并获得标准输出、标准错误与退出状态。
- 允许模型递归匹配文件路径，同时跳过常见大型或生成目录。
- 允许模型用正则搜索文本文件，并跳过二进制文件与无关目录。
- 允许模型检索并激活延迟工具。
- 允许模型暂停执行，在终端向用户提出结构化问题后继续。
- Anthropic 与 OpenAI 协议都能发送工具定义、解析流式工具调用并回传工具结果。
- 在一次用户输入内循环处理一个或多个工具调用，直到模型给出最终文本或执行失败。
- 在 TUI 中可观察模型的工具调用、工具结果、错误和最终回答。

## 4. 非功能要求

- 工具输出必须设置统一上限，防止单次结果挤占上下文或拖慢终端渲染。
- 只读工具可被后续调度器并行化，写入与命令工具必须保留串行语义。
- 命令执行必须并发消费标准输出与标准错误，避免子进程管道阻塞。
- 命令执行必须支持超时、强制终止和线程中断恢复。
- 文件工具只在当前 YuanCode 工作目录范围内操作，规范化路径后拒绝目录逃逸。
- 文件读取、写入与 Provider JSON 序列化统一使用 UTF-8。
- 文件扫描必须跳过版本库、虚拟环境、依赖目录和常见缓存目录。
- 工具 schema 与工具列表保持确定性顺序，保证请求可测试、日志可比较。
- 会话中的工具调用参数、调用标识、工具结果和错误状态必须跨轮完整保留。
- Agent Loop 必须设置最大迭代保护，避免模型无限调用工具。
- 工具失败应作为可回传给模型的结果处理；协议、网络或循环级失败才终止本轮。
- 所有核心能力必须有无网络单元测试，Provider 映射与完整循环必须有自动化集成测试。

## 5. 设计骨架

### 5.1 工具层

工具层由统一工具契约、执行类别、结果对象和注册表组成。每个工具自行提供描述与输入结构，但不感知具体 LLM Provider。注册表负责稳定顺序、协议格式转换和延迟发现状态。

六个默认工具分为三类：文件读取与搜索属于只读类，文件写入与编辑属于写入类，Bash 属于命令类。终端询问和工具搜索作为扩展工具注册。

### 5.2 会话与协议层

会话消息除了文本与思考内容，还能表达 assistant 发起的工具调用和 user 回传的工具结果。Anthropic 与 OpenAI 客户端分别把统一消息和工具 schema 转换为各自请求格式，并把流式工具参数增量还原成统一事件。

### 5.3 执行层

执行器根据注册表定位工具，校验未知工具，执行并统一截断输出。路径型工具以启动时确定的工作目录为边界。单个工具的业务失败被封装为工具结果，以便模型理解失败原因并修正下一步动作。

### 5.4 最小 Agent Loop

每次用户输入先写入会话并请求模型。若模型返回文本，则继续流式渲染；若返回工具调用，则保存 assistant 调用、执行工具、展示结果并把结果写回会话，然后带着更新后的历史再次请求模型。循环在模型正常结束、发生不可恢复错误、用户取消或达到迭代保护时停止。

### 5.5 TUI 交互

现有欢迎卡片和输入框保持不变。工具调用以独立、紧凑的状态行展示；AskUser 在当前终端读取用户选择或拒绝；工具完成后模型最终回答继续沿用现有 Markdown 流式渲染。

## 6. Out of Scope

- 不实现权限审批、危险命令确认和可配置 allowlist/denylist。
- 不实现工具调用的并行调度，本阶段按模型返回顺序串行执行。
- 不实现 MCP、Hook、Subagent、Worktree、TodoList、Team 或后台任务工具。
- 不实现文件补丁协议、语法树编辑、Git 专用工具或 IDE 集成。
- 不实现命令会话复用、交互式子进程或长期后台进程管理。
- 不支持 PowerShell、cmd 或其他命令解释器，命令工具固定调用 Bash。
- 不实现多模态工具输入与二进制文件编辑。
- 不实现持久化会话恢复，工具相关历史仅保存在当前进程内。
