# YuanCode v0.4 规格：蓝色卡片式终端展示层

## 背景

YuanCode 第一阶段已经打通 Java 命令行启动、配置加载、Anthropic/OpenAI/DeepSeek Provider、SSE 流式回复和进程内多轮对话，但当前交互仍是简单的 `you>` / `assistant>` REPL。它能证明通信链路，却缺少成熟 Coding CLI 应有的输入编辑、信息层级、流式状态和终端适配。

本阶段将纯对话外壳升级为 Claude Code 风格的内联终端界面：保留终端滚动历史，在当前输入区域进行编辑和局部刷新，而不是构建占满屏幕的传统 TUI。阶段完成后再单独接入工具调用和 Coding Agent 能力。

## 目标用户

- 在 Windows Terminal 与 PowerShell 7 中使用 YuanCode 的开发者。
- 需要多行编辑、输入历史和清晰 Markdown 输出的终端用户。
- 后续将在稳定终端外壳上开发 Coding Agent 能力的维护者。

## 目标

- 让 YuanCode 在信息层级、布局节奏和交互反馈上接近当前 Claude Code 的内联终端体验，同时保留 YuanCode 自有名称、字标和文案。
- 在不破坏现有 Provider 与会话能力的前提下替换简单输入输出层。
- 让欢迎页、消息、状态、错误和系统提示形成清晰且一致的视觉层级。
- 支持多行输入、终端编辑快捷键、运行期输入历史和可靠取消。
- 支持流式 Markdown 子集，并避免因整段反复重绘造成闪烁或重复输出。
- 在无 ANSI、重定向输出和非交互环境中自动降级为稳定的纯文本模式。

## 能力清单

- 启动时展示蓝色圆角字符卡片、紧凑 Yuan 图标、版本、当前模型、Provider 和工作目录。
- 启动卡展示帮助提示，卡片边框和内容在宽窄终端中保持对齐。
- 欢迎页在终端宽度不足时自动切换为纵向紧凑布局。
- 用户输入以强调前缀展示，助手正文不再显示 `assistant>` 标签。
- 用户可使用常见导航、删除、按词移动和行首行尾编辑操作。
- 输入区上方展示当前系统用户与 YuanCode 身份，输入内容位于独立蓝色圆角字符框中。
- 输入区使用醒目的单字符主提示符、细线续行提示和不抢占正文的快捷帮助提示。
- 思考过程和助手回答使用不同颜色的圆点建立视觉层级，无颜色模式仍保留圆点。
- 用户可输入多行消息，并通过独立快捷键插入换行。
- 多行粘贴被视为一条消息，不会在粘贴过程中自动发送。
- 用户可通过上下方向键浏览当前进程中的输入历史。
- 输入历史只保存在当前进程，退出后不落盘。
- 流式回复开始前显示动态等待状态，并展示已经等待的时间。
- 收到正文后立即流式展示，不等待完整响应。
- 完成后展示耗时和可用的 token 用量。
- 普通 Markdown 文本、标题、列表、强调、行内代码和代码块具有可辨识样式。
- 代码块结束前以稳定的流式形式输出，结束后不重复打印整条回复。
- 窗口大小变化会影响后续绘制，但不会重写已经进入终端历史的内容。
- 长回复依赖终端自身滚动，不引入应用内分页器。
- 生成期间按中断键会取消当前请求、保留屏幕上的部分输出但不写入会话历史。
- 空闲输入区按 EOF 或执行退出命令会正常退出并恢复终端状态。
- 配置禁用颜色、终端不支持样式或输出被重定向时自动使用纯文本表现。
- 用户可查看当前模型、Provider、工作目录、会话轮数和配置文件路径。
- 配置查看命令只显示路径，不输出配置内容或认证信息。
- 原有清空会话、切换 Provider、帮助和退出命令继续可用。

## 非功能要求

- 使用 Java 21，并通过 JLine 提供终端、行编辑、信号和尺寸抽象。
- Windows 首要验证环境为 Windows Terminal 与 PowerShell 7，其他 ANSI 终端采用能力检测适配。
- 输入层、Markdown 渲染、流式状态和业务会话保持独立边界，Provider 层不依赖终端库。
- 交互终端创建失败时必须自动退化为纯文本模式，而不是阻止对话功能启动。
- 所有终端模式、光标状态、颜色和信号处理器在正常退出、异常和取消后均应恢复。
- 动态状态刷新不得阻塞 SSE 消费或用户中断处理。
- 已打印到终端历史的回复不得因重排或完成态渲染而重复出现。
- 界面颜色必须提供无颜色等价信息，不以颜色作为唯一语义载体。
- 标识和布局必须是确定性字符输出，不依赖图片、字体文件或网络资源。
- 自动化测试使用虚拟终端和本地伪 SSE 服务，不依赖真实 API Key。

## 设计骨架

本阶段在现有配置、会话和 Provider 之上新增四个界面边界：

1. 终端会话边界负责建立 JLine system terminal、信号处理、能力检测、尺寸读取、资源恢复以及纯文本降级。
2. 输入编辑边界负责多行缓冲、提交、换行快捷键、粘贴保护、进程内历史和 EOF 语义。
3. 展示边界负责主题、欢迎页、Yuan 字标、消息前缀、系统提示、错误与状态行。
4. 内容渲染边界负责增量 Markdown、代码块状态和不重复输出策略。

交互主流程继续负责解析内置命令、调用统一 LLM 客户端以及在完整结束后写入会话。流式展示只消费统一事件，不理解 Anthropic、OpenAI 或 DeepSeek 的协议细节。

## Out of Scope

- Tool Use、函数调用、Shell 执行和权限确认。
- Claude 名称、Claude 图标、Anthropic 专有文案或让用户误认为本项目是官方 Claude Code 的品牌元素。
- 仅为模仿截图而展示不可操作的工具调用卡、权限弹窗或 Agent 状态。
- 文件读取、`@文件` 引用、路径补全和代码编辑。
- Agent Loop、任务规划、SubAgent、Team 和多智能体协作。
- Git 状态、diff 展示和版本控制操作。
- 会话落盘、输入历史落盘和退出后的恢复。
- 全屏 TUI、固定消息面板、鼠标操作和应用内滚动条。
- Markdown 表格、图片、HTML、数学公式和完整 CommonMark 兼容。
- 终端内真正可点击或交互式折叠区域。
- 旧版 `cmd.exe` 的完整交互与视觉兼容。
- 费用估算、账号用量和 API 账单展示。
- 文件路径或命令的自动补全。
