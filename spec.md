# ch04：异步 Agent Loop Spec

## 1. 背景

YuanCode v0.5.0 已经具备最小 ReAct 循环：模型可以调用工具，工具结果会回填会话，随后继续请求模型直到得到最终文本。但当前循环仍由 TUI 线程同步驱动，过程通过 Listener 回调暴露；同轮多个工具全部串行执行，也没有独立的 Plan-only 状态或运行级取消边界。

这会带来三个问题：上层界面与 Agent 内部状态机耦合；多个安全读取无法并发；取消或超时时容易在会话中留下未完成的轮次。本阶段把现有最小循环升级为异步、事件驱动、可取消且支持读写分组的 Agent Loop。

## 2. 目标用户

- 希望在终端中观察模型思考、工具执行和最终回答全过程的 YuanCode 用户。
- 希望只让模型分析项目并给出计划、不允许修改文件或执行命令的用户。
- 后续需要为 TUI、CLI、权限系统或其他上层复用 Agent 事件流的维护者。

## 3. 能力清单

- 每次用户请求由独立后台运行驱动，不阻塞终端输入与渲染线程。
- Agent 对外输出统一事件流，覆盖用户消息、思考、文本、工具开始、工具结果、用量、轮次完成、最终回复、循环完成、模式变化和错误。
- ReAct 状态机在模型无工具调用、显式结束、用户取消、超时或达到循环上限时按明确原因终止。
- 同一轮中的只读工具可以并发执行，写入与命令工具保持串行执行。
- 并发工具结果可以按完成时间展示，但回填会话时保持模型原始调用顺序。
- Plan-only 模式只允许只读工具，写入与命令工具被拦截并作为工具错误回传模型。
- 用户可通过终端命令切换、开启或关闭 Plan-only 模式，并在状态信息中观察当前模式。
- 取消操作会停止活动模型流、等待中的工具任务和 Agent 后台线程。
- 流式事件静默超时会结束当前运行，并向上层输出可观察错误。
- 运行取消或失败时不会把不完整的 assistant 工具调用或部分工具结果留在会话历史中。
- 为后续权限审核、Hook、上下文压缩和重试策略保留清晰的执行边界，但本阶段不实现这些策略。

## 4. 非功能要求

- Agent 运行必须使用 Java 21 虚拟线程，TUI 只消费事件队列。
- 事件队列必须有界，并在消费者停止时能够响应线程中断，避免后台线程永久阻塞。
- READ 工具并发不得改变会话中工具结果与调用标识的对应关系。
- WRITE 和 COMMAND 工具不得并发，避免文件覆盖、命令副作用交叉或输出顺序不确定。
- Plan-only 的限制必须在执行器边界强制实施，不能仅依赖提示词约束模型。
- 每次运行只有一个明确终态，完成、取消、错误和达到上限不能重复发出。
- 取消和失败后的 Conversation 必须回到本次运行开始前的一致检查点。
- 工具输出继续遵守统一长度上限，避免下一轮上下文膨胀。
- 新事件模型、并发执行、Plan-only 和取消语义必须通过无网络自动化测试验证。
- 现有 Provider、工具、Markdown 渲染、多轮对话和 Ctrl+C 行为不得回归。

## 5. 设计骨架

### 5.1 运行句柄

一次用户请求对应一个 Agent 运行句柄。句柄持有有界事件队列、取消入口和完成状态；后台虚拟线程负责推进状态机。上层可以阻塞或定时读取事件，但不直接参与模型流消费和工具调度。

### 5.2 Agent 事件模型

LLM 的 Provider 事件仍属于协议层；Agent 将其转换为面向产品流程的事件。工具调用事件在真正开始本地执行时携带完整参数；工具结果事件包含调用标识、工具名、结果、错误状态和耗时。最终回复与循环完成分离，使上层既能渲染正文，也能可靠恢复 idle 状态。

### 5.3 ReAct 状态机

每轮依次经历请求模型、消费流、判断工具调用、执行工具、写回结果和决定下一轮。模型未请求工具时生成最终回复并结束；请求工具时先完成整批执行，再将 assistant 工具调用和有序结果原子写入会话，随后进入下一轮。

### 5.4 分组执行

执行器按工具类别把调用分为只读组和副作用组。只读组使用虚拟线程并发执行，并立即推送完成事件；副作用组按照模型原始顺序串行执行。全部任务结束后，根据原始调用索引重排结果，再回填 Conversation。

### 5.5 Plan-only

Plan-only 是 Agent 的运行状态，而不是单纯的提示文本。模式开启时，工具 Schema 仍可让模型看见完整能力，但执行器只放行 READ；WRITE 与 COMMAND 返回明确的受限结果。最终回答应是一份供用户审批的计划，关闭模式后用户可要求执行。

### 5.6 取消与一致性

运行开始时记录 Conversation 检查点。取消会同时关闭活动流、中断后台线程并取消尚未完成的工具任务。只有完整模型轮次和完整工具批次才能提交会话；运行异常或取消时恢复到检查点，并发出唯一终态事件。

## 6. Out of Scope

- 不实现具体权限策略、危险命令确认或永久授权规则。
- 不实现 PreToolUse、PostToolUse 或其他 Hook 引擎。
- 不实现上下文压缩、自动摘要或 token 预算管理。
- 不实现 rate-limit 退避、context-too-long 重试或 max_tokens 续写恢复。
- 不实现复杂 system prompt 组装，仅保留最小 Plan-only 提醒。
- 不实现 Subagent、Agent-as-Tool、任务委派或递归 Agent Loop。
- 不实现后台任务、跨进程恢复或持久化运行队列。
- 不实现 Plan 文件编辑器或审批对话框；本阶段只生成文本计划并由用户显式关闭模式后执行。

## 7. 完成定义

当 [checklist.md](checklist.md) 的全部适用条目均可勾选，Maven 全量测试与打包通过，并完成异步事件、并发读取、Plan-only 和取消的端到端验证，即认为 ch04 完成。
